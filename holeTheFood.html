Level 27 featuring bouncy fruit.
Level 17 with cube watermelons, oranges and apples.

Overview

Hole the Food was created within a few weeks as a summer project with the aim of learning about developing for mobile platforms using the Unity game engine. Ads inserted in the game played every few levels, the ads were added in created using Unity Ads. The game was not commercially successful but this was not the goal. 50 levels were created for the game.

The game's map tilting is entirely controlled by the buttons on the bottom left of the screen, the player can choose 8 degrees of movement (up, up-right, up-left etc). The number of stars achieved in each level was determined on how fast the player is able to complete the level in. The player can also quickly restart the level by pressing the top right arrows. If a food object falls off the map and not the green trigger square, the player must restart the level. Each food has different physics attributes, some are more bouncy and some slide/roll more.


Tools used

- Unity URP for creating the game.

- Photoshop for thumbnails.

- Blender for 3D modelling and some texturing.

- Adobe Premiere Pro for creating the trailer.

- GitHub for version control.


What I learnt

- Learnt about the regulatory rules surrounding Google Store and Apple Store mobile development such as App Tracking Transparency (ATT).

- Learnt about the development process of a mobile game using the Unity Game Engine and how to use Unity ads.

- How to manage creating a small scoped project in a short timeframe.

- The hypercasual game market is very hard to break into if the aim is to make a commercially viable product, it is an extremely competitive and saturated market compared to other genres and platforms.

- Ad monetisation generates very little revenue per person compared bought software, even 1000 ad impressions can make as little as $5 (and many users will not get any impressions if they do not play to the point of getting an ad and/or they play offline).
