Overview

This project was developed with a team of 5 people for the 2nd year Software Development module at University of Sussex. The aim of the project was to recreate the Settler’s multiplayer board game into a video game that closely aligned with the 'spirit' of the board game. I led the weekly end/start of agile sprints, backlog refinement and helped team members with bugs and issues they had encountered. I programmed some of the building mechanics, player customisation (colour, name, portrait selection) the tile number generation, the robber and the menus, putting the board together using materials, provided textures, found textures that I am licensed to use (commercial use, public domain, asset packs etc) and put together assets to create the room and the lighting. Project achieved a total of 85/100 (unfortunately only a partially functional AI was created in the strict timeframe given for the project).

Tools used

- Unity Universial Render Pipeline (Unity URP).

- Blender to create the 3D assets (settlements, cities, robber, dice, card objects).

- Hack n Plan for sprint cycle planning.

- GitHub for version control.

- Draw.Io for UML diagrams.

- OBS to record gameplay.


The Settlers Features:

- A functional system to place settlements according to the games rules.

- A functional system to trade between players.

- A functional system for turn taking.

- The usage of event and development cards such as Knight cards and Year of Plenty.

- Generation of resources from tiles the player's settlements are adjacent to.

- Movement of the robber by an individual player.


What I learnt

- In depth about how scrum and agile works.

- Teamwork and leadership skills in a medium scoped project.

- How lighting and baking works in Unity URP.

- It is very important to regularly build Unity projects because many bugs that appear may not be encountered in the editor.
